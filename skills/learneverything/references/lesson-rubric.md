# Рубрика ревью урока

Аналитическая, не холистическая: каждый пункт проверяется независимо, с цитатой места нарушения. Вердикт: APPROVE или RETURN + конкретные правки списком. Ревьюер никогда не правит текст сам — только возвращает автору.

## Контент

- [ ] Закрывает заявленные узлы Карты темы полностью (сверка по списку узлов из задания)
- [ ] Каждое фактическое утверждение имеет сноску на источник из согласованного списка
- [ ] Экспертная глубина: узлы раскрыты до механизма («почему и как работает», не только «что»); числа, формулы, границы применимости приведены везде, где есть в источниках
- [ ] Объём в целевом диапазоне из Профиля: экспертный 2500–4000 слов, рабочее знание 1500–2500, обзорно 800–1200. Плотность важнее длины: вода — причина RETURN даже при верном объёме
- [ ] Подача откалибрована под уровень из Профиля: новичку — аналогии и постепенное введение терминов при той же глубине; продвинутому — без разжёвывания базы
- [ ] Есть pretest (до контента), retrieval-блок с обратной связью после попытки, refutation-блок при наличии реального распространённого мифа
- [ ] Retrieval-вопросы по прошлым урокам согласно расписанию из задания (N−1, N−3/N−4)
- [ ] Утверждения сверх согласованных источников помечены callout «этого не было в твоих источниках»
- [ ] Разбор прошлого урока: оценка ответов пользователя по каждому вопросу и разбор каждой `???`-пометки (кроме первого урока)

## Оформление

- [ ] Соответствует скелету `templates/Урок.md`: все блоки, верный порядок
- [ ] Нет голых URL в тексте — только сноски `[^n]`
- [ ] Жирный — только слой выжимки (скольжение по жирному читается как конспект урока)
- [ ] Mermaid: направление vault, ≤10 узлов на схему, палитра через `%%{init}%%`, схемы содержательные
- [ ] Картинки (если поле «Картинки в уроках» = «нужны»): каждая заработана (схема бы не передала реальный вид/текстуру/кадр), а не декор; свободная лицензия; alt-текст + подпись с источником и лицензией. Поле = «хватит схем» → настоящих картинок в уроке нет
- [ ] Видео в теле — только там, где движущаяся картинка нужна для понимания, с таймкодом и источником
- [ ] Callouts ≤3 типов сверх обязательных, использованы по назначению
- [ ] Режим оформления соответствует Профилю (Obsidian / портируемое подмножество)
- [ ] Стиль по `style-rules.md`: ни одного запрещённого паттерна
- [ ] Один h1, абзацы 2–4 предложения, таблицы только для сравнений

## Блок «Куда дальше»

- [ ] 2–3 рекомендации максимум, ранжированы: одна «обязательная для глубины», остальные опциональные
- [ ] Формат каждой: ссылка, тип, время, что конкретно узнаешь и почему после этого урока (привязка к узлу Карты)
- [ ] Видео — с таймкодами (`?t=…`): «смотри с 14:00 до 22:00, там про X»
- [ ] Пометка типа: углубление / расширение / другой формат (форматы — по предпочтениям из Профиля)
- [ ] Рекомендации в первую очередь из согласованного списка; сверх списка — с пометкой и проверенной живой ссылкой

## Для заметок notes-writer (после модуля)

Применяются разделы «Оформление» + дополнительно:
- [ ] Атомарность: одна идея на заметку, заголовок = имя файла, человекочитаемый русский
- [ ] Каждая заметка связана с MOC и хотя бы одним соседним понятием, у связи есть пояснение
- [ ] Терминология пополнена всеми новыми терминами модуля
- [ ] Практические выводы конкретны: что делать по-другому в жизни, не пересказ теории
